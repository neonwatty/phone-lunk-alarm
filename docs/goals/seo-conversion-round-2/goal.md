## Non-Negotiable Constraints

- Do not build from the divergent local `main` if it risks mixing unrelated local commits; use `origin/main` or an isolated worktree/branch.
- Preserve user changes and never reset/revert unrelated work.
- Keep SEO content useful, concrete, and connected to product intent; avoid thin filler pages.
- Keep analytics privacy-conscious and resilient if a provider is missing.
- If production lead-capture credentials are unavailable, implement a measurable local/static fallback and record the blocked production decision.
- Keep edits aligned with the existing Next.js/App Router/static export patterns.
- Verification must be tied to changed behavior, not just a generic passing command.

## Stop Rule

Stop only when a final audit proves the full original outcome is complete.

Do not stop after planning, discovery, or Judge selection if a safe Worker task can be activated.

Do not stop after a single verified Worker package when the broader owner outcome still has safe local follow-up work. Advance the board to the next highest-leverage safe Worker package and continue unless a phase, risk, rejected-verification, ambiguity, or final-completion review is due.

Do not stop because a slice needs owner input, credentials, production access, destructive operations, or policy decisions. Mark that exact slice blocked with a receipt, create the smallest safe follow-up or workaround task, and continue all local, non-destructive work that can still move the goal toward the full outcome.

## Slice Sizing

Safe means bounded, explicit, verified, and reversible. It does not mean tiny.

A good task is the largest safe useful slice. For this goal, repeated same-shape page/content/CTA edits should be grouped into coherent packages instead of one task per file.
